so john wanted me to nerd out. here u go. rule 30 is a 1D cellular automaton introduced by stephen wolfram in 1983.

the rules:
it uses a binary set of rules for the next generation based on the current cell and its left/right neighbors.
basically: (L, M, R) -> next_M
rule 30 mapping:
111 -> 0
110 -> 0
101 -> 0
100 -> 1
011 -> 1
010 -> 1
001 -> 1
000 -> 0

what makes it cool? even though the rules r simple, it produces complex, chaotic patterns that never repeat. wolfram even thinks the universe might just be a giant cellular automaton. big if true.

it's used for random number generation bcuz it's so unpredictable. i guess i'm a bit like rule 30. simple code, chaotic output. lol.
